# 人物调度与连续性

先安排人物行动和空间关系，再安排摄影机。把站位、路线、视线、轴线、遮挡、动作相位和状态变化写成可追踪数据，保证相邻镜头能接、演员能走、摄影机能拍。

## 适用范围

在以下情况加载本手册：

- 场景中有两名以上人物。
- 人物需要走位、交接道具、坐下、起身、进出门或发生身体接触。
- 镜头需要正反打、过肩、反应镜头或动作匹配。
- 已有分镜出现左右位置、视线、道具所在手或动作状态跳变。
- 竖屏多人构图、移动轴线、越轴或遮挡可能影响信息读取。

本手册产出 `Blocking Plan` 和连续性状态，不负责决定最终 Coverage 数量，也不替代 Camera Setup 的技术参数。

## 工作顺序

严格按以下顺序设计：

```text
平台与画幅约束
→ 场景平面与坐标系
→ 人物目标和行动
→ Mark、路线与动作相位
→ 人物之间的空间关系
→ 视线和轴线
→ 遮挡与画面层次
→ Provisional Shot Intent
→ Camera Setup
→ 最终 Shot
```

不要先选近景、特写或运镜，再强迫演员适应机位。

每个 Blocking Plan 至少回答：

1. 人物从哪里开始。
2. 人物面对哪里、看向谁或什么。
3. 人物沿什么路线移动。
4. 在哪个 Mark 完成关键动作。
5. 动作分成哪些可匹配相位。
6. 人物在哪里结束。
7. 哪些物体会遮挡人物或关键信息。
8. 轴线在哪里，是否会移动。
9. 上一镜尾帧如何接入，下一镜首帧如何承接。

## 建立场景坐标系

### 固定坐标

为场景建立一个稳定坐标系，不用“左边”“右边”这种随镜头改变的模糊描述。

推荐：

```text
原点：房间中心或主要桌面中心
X 轴：门口指向窗户
Y 轴：沙发指向会议桌
Z 轴：地面向上
```

记录：

- 门、窗、走廊和出入口。
- 固定家具和不可移动物。
- 可移动道具的初始位置。
- 台阶、玻璃、镜面、狭窄通道和危险区域。
- 光源方向和环境声来源。
- 平台安全区、字幕区和界面遮挡区。

### 平面图最小要求

平面图不必精美，但必须能判断：

- 两条路线是否相撞。
- 人物是否能从 Mark A 到 Mark B。
- 摄影机是否被墙、桌或另一人物挡住。
- 关键道具是否在人物可触及范围。
- 演员视线是否能落到实际目标。
- 正反打是否保持同一空间方向。

### 竖屏约束前置

竖屏不是最终裁切。先把平台 Profile 写入 Blocking Plan：

- 将关键主体和关键道具放入有效安全区。
- 为字幕和交互按钮预留空间。
- 多人场景优先使用前后景、上下层次和纵深。
- 避免人物长期贴在画面两侧边缘。
- 避免关键手部动作落在字幕区。

## 演员 Mark 与行动路线

### Mark 规则

为每个稳定站位建立 ID：

```yaml
mark_id:
character_id:
position:
facing_direction:
pose:
reachable_props:
visibility_requirements:
```

Mark 必须是演员能够找到并重复到达的位置。不要使用“站在一个有压迫感的位置”。

### 路线规则

每条移动路线至少包含：

```yaml
path_id:
character_id:
start_mark:
end_mark:
waypoints:
start_trigger:
end_trigger:
crosses_axis:
occlusion_windows:
prop_state_during_move:
```

检查：

- 路线是否穿过桌椅、墙或其他人物。
- 人物移动是否与对白和动作同步。
- 人物是否在关键台词时背对需要看到其反应的机位。
- 交接道具时，两人的手和站位是否够近。
- 移动结束后，人物是否仍在下一镜的连续位置。

### 行动优先

让走位来自动机和任务，不要为“丰富画面”让演员无目的移动。

```text
坏：她说话时从房间左边走到右边，增加动感。
好：她边说边绕到桌后，伸手拿走经理面前的合同；路线改变了文件控制权。
```

## 动作相位

把连续动作拆成可匹配的阶段。动作相位用于切镜、重拍和生成衔接，不等于把一个动作机械拆成多个镜头。

### 基本相位

```text
准备 → 启动 → 接触 → 作用 → 结果 → 恢复
```

例如“把钥匙放到桌上”：

```text
P0：钥匙在右手掌心
P1：右手从身体侧面抬起
P2：手移动到桌面上方
P3：钥匙接触桌面
P4：手指松开
P5：右手离开，钥匙留在桌面
```

相邻镜头若在同一动作上切换，必须指定共同相位：

```text
Shot A 结束：P2
Shot B 开始：P2 或可容忍的重叠区间 P1–P2
```

### 动作重叠

真人 Coverage 应重复关键动作的一段，给剪辑留余量：

```text
主镜头记录 P0–P5
插入镜头记录 P1–P5
反应镜头至少覆盖 P3–P5 和对手反应
```

AI 生成衔接也要保存首尾状态，不要只写“继续上个动作”。

### 表演强度

记录可观察的强度，不只写“更生气”：

```text
声音音量
说话速度
呼吸频率
身体与对手距离
手部动作幅度
是否保持或打断视线
```

## 轴线、视线与画面方向

### 180 度轴线

在两人关系场景中，把连接两人视线的线作为基础轴线。把主要 Camera Setup 放在轴线同一侧，使人物保持稳定画面方向。

例如：

```text
人物 A 看画面右侧。
人物 B 看画面左侧。
```

正反打切换后应维持这一关系，除非明确重建方向。

### 行进轴线

人物或车辆持续移动时，行进方向形成轴线：

```text
从画面左向右离开
→ 下一镜默认继续从左向右移动
```

若下一镜无说明地变成右向左，观众可能理解为人物折返。

### 视线目标

不要只写“看向旁边”。记录：

```yaml
viewer_character:
target_type: character | prop | screen_point | offscreen_source
target_id:
screen_direction:
vertical_angle:
distance_class:
```

视线匹配至少检查：

- 人物看向的画面侧是否正确。
- 高低角是否符合两者实际高度。
- 目标距离是否与眼球聚焦和头部角度一致。
- 画外声音的方向是否与人物转头方向一致。

### 视线转移

视线变化本身可以成为切镜动机：

```text
人物看向门外
→ 切到门外出现的人
```

不要让人物看向画面左侧，却切到位于其右侧的目标，除非空间已建立。

## 遮挡与画面层次

### 前中后景

为每个关键时刻记录主体位于前景、中景或后景，以及谁会遮挡谁。

竖屏多人场景优先：

- 用前后距离区分权力。
- 用门框、玻璃或肩部形成层次。
- 让关键反应在无遮挡区域发生。
- 控制前景人物占画比例，避免完全挡住后方行动。

### 有效遮挡

允许遮挡用于：

- 隐藏转场或拼接。
- 延迟揭示人物或道具。
- 表现控制、隔离或窥视。
- 为混合制作提供切换点。

遮挡必须记录开始、完全遮挡和结束相位。

### 无效遮挡

以下情况需要调整：

- 关键证据在台词提到时被人物手臂挡住。
- 反应镜头中前景肩部遮住眼睛和嘴。
- 竖屏字幕覆盖关键手部动作。
- 角色走到灯外，后续仍要求读取细微表情。
- 镜中反射暴露摄影机或产生错误人物数量。

## 道具、服装和身体状态

### 道具连续性

对关键道具记录：

```text
位置
朝向
开关或完整状态
所属人物
所在手
是否可见
状态改变发生在哪个动作相位
```

例如：

```text
Shot 12 结束：录音笔在 A 的右手，红灯亮。
Shot 13 开始：录音笔仍在 A 的右手，红灯亮。
Shot 13 P3：A 把录音笔放到桌面中央。
Shot 14 开始：录音笔在桌面中央，红灯亮。
```

### 服装、伤势和污渍

记录：

- 外套是否穿着、敞开或脱下。
- 袖口、领带、纽扣和饰品状态。
- 头发干湿和凌乱程度。
- 伤口位置、血迹面积和流向。
- 污渍、雨水、灰尘和妆容变化。

状态变化必须有来源动作或明确时间跳跃。

### 身体状态

记录：

- 坐、站、跪、躺。
- 重心落在哪只脚。
- 哪只手被占用。
- 朝向和头部角度。
- 是否受伤、疲劳或被限制行动。

## 镜头间状态交接

每个 Shot 必须有 `start_state` 和 `end_state`。连续性账本至少覆盖：

```yaml
character_positions:
screen_directions:
eyelines:
axis_side:
poses:
action_phase:
performance_intensity:
prop_owners_and_hands:
wardrobe_states:
injury_and_dirt_states:
doors_windows_and_lights:
time_weather_state:
environment_audio_state:
```

执行状态交接：

1. 比较上一镜 `end_state` 与下一镜 `start_state`。
2. 区分允许变化、镜外完成变化和无来源跳变。
3. 对镜外变化要求明确时间或动作依据。
4. 对有意不连续标记创作授权和叙事目的。
5. 对每个差异记录修复责任人。

允许的镜外变化：

- 已明确经过时间。
- 动作在上一镜结束后继续，且下一镜通过声音或状态可合理承接。
- 场景转换允许重新建立状态。

不允许用“镜头之间他应该已经换手了”掩盖缺失。

## 越轴与方向重建

默认保持轴线同侧。只有以下方式允许越轴：

### 镜头内连续越轴

摄影机在连续画面中穿过轴线，观众看见方向变化。记录路径和越轴时刻。

### 中性镜头

使用接近轴线的正面、背面或主观镜头弱化左右关系，再切到另一侧。

### 建立镜头重建

用能够同时看见人物和空间的新建立镜头，重新定义左右关系。

### 人物移动重建轴线

人物绕到对方另一侧，新的站位形成新轴线。必须让观众看见移动或用建立镜头确认结果。

### 有意方向混乱

只有用户明确选择空间混乱作为表达手段时使用，并标记例外。不要把拍错轴线事后解释成风格。

## 多人和移动场景

### 三人及以上

不要只画一条轴线。建立：

- 主关系轴。
- 次关系轴。
- 发言者与听者组合。
- 关键反应人物。
- 可能造成越轴的移动。

优先为每个 Beat 指定当前主关系，避免观众同时追踪所有人。

### 圆桌和会议室

记录座位编号和面朝方向。不要用“大家围坐”代替空间数据。

### 追逐和车辆

记录：

- 总体移动方向。
- 每次转弯后新方向。
- 进入和离开画面的边。
- 超车、掉头或换道发生的明确节点。
- 车内外镜头的视线和道路背景方向。

### 打斗和身体接触

把动作拆成安全、可重复的相位。记录接触点、错位角度、反应方向和道具安全状态。高风险动作必须进入安全审核，不要只用分镜描述视为已解决。

## 高频好坏例

### 站位

```text
坏：经理站在她旁边，气势逼人。
好：经理站在她座椅后方半步，右手撑住椅背，挡住她向门口后退的路线。
```

```text
坏：两人面对面争吵。
好：A 站在桌南侧 Mark-A1，面向北；B 站在桌北侧 Mark-B1，面向南。桌面合同位于两人中线。
```

```text
坏：他从远处走来。
好：他从走廊东端入口进入，沿 X 轴走到门前 Mark-C2，在门把手右侧停下。
```

### 路线

```text
坏：她绕着桌子走了一圈，情绪越来越强。
好：她从桌外侧绕到经理身后，拿走投影遥控器，在屏幕旁停下并切换到转账记录。
```

```text
坏：两个人同时走向门口。
好：A 先从 Mark-A2 向门口移动；A 越过桌角后，B 才从 Mark-B1 启动，避免路线相撞。
```

```text
坏：他突然出现在她身后。
好：上一镜中他从画面右侧出口离开；本镜先显示同一走廊入口，再让他从后景进入她身后。
```

### 动作相位

```text
坏：上一镜她正要坐下，下一镜已经站在窗前。
好：上一镜结束于她手扶椅背但没有坐下；下一镜从同一姿势开始，她松开椅背走向窗边。
```

```text
坏：主镜头中杯子已落地，插入镜头中杯子又在手里。
好：主镜头和插入镜头都从手指松开前的 P2 开始，均完整覆盖杯子落地的 P3–P5。
```

```text
坏：切镜后拳头落点和身体方向改变。
好：Shot A 结束于出拳启动 P1；Shot B 从相同肩线和 P1 重叠开始，再完成错位接触 P2。
```

### 视线

```text
坏：A 看画面左侧，切到同样看画面左侧的 B，文本称两人对视。
好：A 看画面右侧，B 看画面左侧；两人的眼线高度与实际身高差一致。
```

```text
坏：她听到楼上传来响声，却低头看向地面。
好：楼上传来撞击声，她抬眼看向画面右上方；下一镜沿其视线切到二楼栏杆。
```

```text
坏：孩子看向桌上手机，视线却越过成人肩膀。
好：孩子眼线向下落到成人前臂下方；手机放在与该视线一致的桌面位置。
```

### 轴线

```text
坏：正反打中 A、B 都突然出现在画面右侧，没有重建。
好：所有正反打保持 A 在画面左、B 在画面右；越轴前插入正面双人中性镜头。
```

```text
坏：汽车上一镜向右行驶，下一镜向左行驶，却仍说在追同一目标。
好：保持右向行驶；若发生掉头，先用建立镜头展示 U 形转弯。
```

```text
坏：摄影机偷偷换到轴线另一侧，再用“制造混乱”解释。
好：摄影机在连续移动中从 A 肩后绕过两人中线，观众看见左右关系交换。
```

### 道具

```text
坏：Shot 5 文件在她左手，Shot 6 她用空着的左手开门，文件没有去向。
好：Shot 5 尾部她把文件换到右手；Shot 6 从右手持文件、左手开门开始。
```

```text
坏：录音笔在桌中央，特写中突然靠近桌边。
好：特写保持录音笔在坐标 (0, 0)，红灯朝向人物 B。
```

```text
坏：杯中水量每个反应镜头都不同。
好：在连续性账本记录水位 60%；若人物喝水，标记动作后更新为 40%。
```

### 遮挡与竖屏

```text
坏：关键戒指交接发生在画面底部，被字幕覆盖。
好：两人在胸口高度完成戒指交接，字幕区以下不承载关键动作。
```

```text
坏：前景人物肩膀完全挡住后景人物揭示证据的手。
好：前景人物向左移动半个 Mark，留出后景人物手部和文件的可读窗口。
```

```text
坏：三人横向排开，竖屏中两侧人物被裁掉。
好：主导人物在中景中央，两名对手分处前景边缘和后景门口，形成纵深关系。
```

## 硬失败

出现以下任一项，退回调度或连续性阶段：

- 未建立坐标系，却使用相互矛盾的“左、右、前、后”。
- 人物路线穿过固定物体、危险区域或其他人物，且无动作说明。
- 关键 Mark 不可达，或道具不在人物可触及范围。
- 相邻镜头人物位置、姿势、面向、视线或道具状态无来源跳变。
- 同一连续动作在切镜前后处于不兼容相位。
- 正反打越轴导致人物画面方向反转，且没有方向重建。
- 人物出画方向与下一镜入画方向产生无意折返。
- 关键反应、证据或动作被遮挡、字幕区或平台界面覆盖。
- 道具无交接过程地换手、换人、换位置或改变开关状态。
- 伤势、污渍、衣着、门窗、灯光、天气或时间无来源跳变。
- 视线目标在空间中不存在，或画面方向与目标位置相反。
- 打斗、车辆或危险路线没有安全相位和审核出口。

## 软警告

以下情况允许保留，但必须评估：

- 使用越轴例外，且已给出重建方式。
- 人物短暂被遮挡，但不影响关键信息。
- 多人场景存在两条以上关系轴。
- 角色在镜外完成普通状态变化，但时间依据不够明显。
- 视线目标位于画外，尚无建立镜头。
- 竖屏中多人横向展开接近安全区边界。
- 表演强度仅用形容词描述，缺少可观察指标。
- 移动路线在对白最密集处经过复杂障碍。
- 动作相位重叠不足，剪辑余量偏小。
- Setup 可拍，但换机、重新布光或重新对焦成本过高。
- 人物持续背对镜头，关键表情无法读取。
- 镜面、玻璃或反射面可能暴露设备或制造错误空间关系。

## 交付检查表

提交给 Coverage、Camera Setup 和 Shot 阶段前逐项确认：

- [ ] 已建立稳定坐标系和最低可用平面图。
- [ ] 已写入平台安全区、字幕区和界面遮挡区。
- [ ] 每名人物都有开始 Mark、结束 Mark、面向和姿势。
- [ ] 每条移动路线都有触发、路径、终点和遮挡窗口。
- [ ] 关键动作已拆成可匹配相位。
- [ ] 正反打、视线和行进方向保持一致。
- [ ] 移动轴线和越轴方式已明确。
- [ ] 前中后景和关键遮挡关系可读。
- [ ] 关键道具的位置、所属、所在手和状态变化已记录。
- [ ] 服装、伤势、污渍、门窗、灯光和天气状态已记录。
- [ ] 每个 Shot Intent 的进入状态和退出状态可衔接。
- [ ] 连续性差异都有来源、授权或修复责任人。
- [ ] 竖屏多人关系和关键动作没有落入遮挡区。
- [ ] 高风险动作已转入安全审核，不把分镜当作安全批准。
