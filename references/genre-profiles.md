# 短剧题材、爽点与节奏 Profile

根据题材选择冲突证据、Beat 密度、台词方式、Coverage 和生成风险，不要给所有短剧套同一套“打脸反转”。把题材 Profile 视为可覆盖的生产配置，不把它当作改写剧情的授权。默认保持人物身份、关系、知识状态、关键证据、事件因果和结果不变。

## 适用范围

在以下情况加载本手册：

- 需要把剧本转换成当前短剧或剧情短视频风格的分镜。
- 需要优化爽点、节奏、台词、钩子和段尾。
- 需要为不同题材选择道具、动作证据和 Coverage。
- 需要判断某种反转是否符合题材逻辑。
- 需要评估 Seedance 在多人、古装、特效、文字、手部或复杂场景中的风险。

不要仅凭题材名称新增套路事实。例如“逆袭爽剧”不自动允许隐藏豪门身份，“家庭伦理”不自动允许出轨或血缘反转。

每个 Profile 至少记录：

```yaml
payoff_patterns:
generation_risks:
fallback_patterns:
prohibited_assumptions:
content_sensitivity_tags:
source_ids:

```

### rhythm_parameters

使用可计算字段：

```yaml
max_consecutive_exposition_beats:
max_unanswered_hook_beats:
minimum_state_changes_per_segment:
reaction_hold_range_seconds:
evidence_read_hold_range_seconds:
dialogue_density_target:
silence_budget_range_seconds:
```

这些数值由项目、平台和测试结果覆盖。Profile 未提供可靠数值时使用 `unknown`，不要编造行业标准。

### dialogue_parameters

至少记录：

```yaml
preferred_turn_length:
maximum_information_units_per_turn:
interruption_allowed:
overlap_allowed:
register:
subtext_level:
exposition_tolerance:
required_read_test:
```

### Profile 新鲜度

题材经验变化慢于平台规则，但仍应版本化。更新 Profile 时运行回归测试，不静默改变已批准项目的节奏和台词。

## 选择与叠加规则

### 选择主 Profile

根据主要叙事发动机选择一个主 Profile：

- 关系选择推动故事：都市情感。
- 权力或资源反转推动故事：逆袭爽剧。
- 未知威胁和证据链推动故事：悬疑惊悚。
- 家庭责任和利益分配推动故事：家庭伦理。
- 组织权限、交易和规则推动故事：职场商战。
- 预期与现实错位推动故事：喜剧误会。
- 身份秩序、礼法与政治风险推动故事：古装权谋。
- 世界规则和技术代价推动故事：奇幻科幻。

### 选择辅助 Profile

最多选择两个辅助 Profile，并记录用途：

```yaml
primary_genre_profile_id:
secondary_genre_profile_ids:
secondary_usage:
conflict_resolution_priority:
```

例如：

```text
主：职场商战
辅：悬疑惊悚
用途：合同证据采用悬疑式延迟揭示
```

当规则冲突时，以主 Profile、事实保真、平台限制和安全要求为优先。

### 不要标签堆叠

不要同时声明：

```text
都市 + 甜宠 + 逆袭 + 悬疑 + 权谋 + 科幻 + 喜剧
```

却没有明确主叙事发动机。标签过多会导致钩子、台词和节奏互相冲突。

## 爽点的可拍定义

把爽点定义为：

> 人物通过可见行动、证据、选择或规则，使先前建立的压制、误解、风险或不公平发生可验证的状态变化。

爽点至少包含：

```text
先前状态
+ 人物行动
+ 可见证据
+ 对方或环境结果
+ 新状态
```

可拍爽点：

- 被否认的授权在系统中验证通过。
- 被抢走的发言权通过投票结果回到人物手中。
- 诬陷被录音、合同或时间戳推翻。
- 人物拒绝对方条件并离开，对方失去合作机会。

不可拍空话：

- “她终于夺回了尊严。”
- “所有人都被她的气场震慑。”
- “这一刻她不再是过去的自己。”

把尊严、震慑和成长转成动作与结果。

### 爽点不是无条件羞辱

不要自动添加：

- 群体下跪。
- 无因扇耳光。
- 路人齐声惊叹。
- 对手突然降智。
- 夸张财富展示。
- 违法报复。

爽点必须来自已建立的能力、证据、规则或选择。

### 爽点强度字段

```yaml
payoff_contract:
  payoff_id:
  setup_beat_ids:
  pressure_state:
  protagonist_action_beat_ids:
  evidence_ids:
  consequence_beat_ids:
  new_state:
  opponent_response:
  authorized_change:
  authorization_id:
  clarity_score:
  causal_score:
  earned_score:
```

若没有 `setup_beat_ids`，反转可能没有力量；若没有 `consequence_beat_ids`，爽点可能只停留在台词。
将此对象写入对应 Hook Contract 的 `payoff_contract`；不要只在自由文本中声称“这里是爽点”。

## 通用节奏设计

### 以状态变化计节奏

不要按固定秒数或固定镜头数判断节奏。统计：

- 人物目标变化。
- 新阻碍出现。
- 新证据出现。
- 权力关系变化。
- 行动产生结果。
- 观众问题被回答或升级。

连续多个 Beat 没有状态变化时，检查是否可压缩。

### 使用节奏段

每个段落可以组合：

```text
建立目标
→ 加入阻碍
→ 人物行动
→ 产生结果
→ 反应或新问题
```

不是每个段落都需要反转。反复反转会削弱因果和可信度。

### 区分加速与混乱

加速：

- 删除重复解释。
- 让动作和对白合法并行。
- 在信息变化处切镜。
- 用反应或 Insert 压缩长台词。

混乱：

- 每句一切。
- 多个钩子同时出现。
- 一个镜头塞入多个地点和结果。
- 对白省略到人物行为没有动机。

## 1. 都市情感

### 叙事发动机

使用关系选择、信任变化、边界建立、承诺兑现或现实利益冲突推动故事。

不要把所有关系戏都写成出轨、误会或豪门身份。

### 常见人物目标

- 确认关系。
- 结束关系。
- 争取解释机会。
- 保护个人边界。
- 在爱情、工作和家庭责任间作出选择。
- 取回共同财物或关键证据。

### 可见证据

优先使用：

- 钥匙、工牌、戒指、合照、行李、租约。
- 已读状态、通话记录、转账、预约。
- 座位距离、让路或挡路。
- 归还物品、删除联系人、拒绝签字。

不要仅靠“眼神复杂”传递关系变化。

### 钩子与爽点

适用钩子：

- 公开选择。
- 关系证据冲突。
- 未完成离场。
- 对方说法与可见记录不一致。

可拍爽点：

```text
对方要求她继续隐瞒关系。
她把共同钥匙放在桌上，拿走自己的行李。
对方追到门口时，门锁已经更换权限。
```

前提是门锁与住所事实来自原稿。

### 节奏

- 给关键选择保留反应时间。
- 不用连续吵架替代关系推进。
- 每轮对话至少改变条件、信任或行动。
- 关系揭示后拍对方的具体行为变化，不只拍惊讶。

### 台词

使用短、直接、带行动条件的台词：

```text
“钥匙还你。”
“你要我留下，就把合同给我看。”
```

避免：

```text
“我曾经那么爱你，可是现在我终于明白我们之间的一切都是虚假的。”
```

若核心事实只是关系结束，可压成动作和短句。

### Coverage

优先：

- 双人关系镜头。
- 独立选择反应。
- 关键物品 Insert。
- 离场方向和门口空间。

### 生成风险

- 亲密接触的肢体连续性。
- 双人对视和口型。
- 手部交换戒指或钥匙。

降级为短动作、独立 Insert、画外对白或真人补拍。

### 禁止推断

- 未说明的出轨。
- 未说明的婚姻或怀孕。
- 未说明的血缘。
- 用昂贵衣服自动证明阶层。

## 2. 逆袭爽剧

### 叙事发动机

使用“压制建立—能力或证据准备—人物主动行动—规则验证—权力状态改变”推动故事。

不要用对手突然变蠢制造逆袭。

### 常见人物目标

- 恢复资格。
- 证明清白。
- 夺回作品、职位、财产或发言权。
- 拒绝不公平条件。
- 让规则按真实权限执行。

### 可见证据

优先使用：

- 授权书。
- 系统权限。
- 投票结果。
- 时间戳。
- 原始文件。
- 录音。
- 交易回执。
- 专业操作结果。

### 钩子

- 压制行为与身份或权限证据并置。
- 对手公开提出条件。
- 关键验证即将执行。
- 人物已准备证据但尚未展示。

### 爽点结构

```text
对手提出可验证指控
→ 主角让系统、文件或第三方规则验证
→ 结果公开
→ 对手失去具体权限或选择
```

正例：

```text
经理说她无权进入会议室。
她刷工牌，门禁显示“董事会临时授权”。
保安移开挡路的手。
```

反例：

```text
她冷笑，所有人立刻意识到她身份不凡。
```

### 节奏

- 尽早建立压制和代价。
- 在兑现前展示主角准备或选择，避免凭空开挂。
- 爽点后产生新状态，不要立即重置同一压制。
- 不连续堆多个身份反转。

### 台词

让强势台词依附可执行动作：

```text
她把投票结果推到屏幕上：“七票。你被停职。”
```

避免无证据宣言：

```text
“从今天起，这里由我说了算。”
```

### Coverage

优先：

- 压制关系镜头。
- 证据 Insert。
- 验证过程。
- 对方动作结果。
- 群体状态变化的单个代表反应。

不要给每个旁观者一个惊讶特写。

### 生成风险

- 屏幕文字。
- 群体同步反应。
- 复杂文件和手部操作。
- 多人口型。

使用独立屏幕 Insert、后期合成文字、少量代表反应和画外对白。

### 禁止推断

- 隐藏首富、神医、战神等身份。
- 无来源的巨额资产。
- 无程序的即时开除或抓捕。
- 违法私刑和羞辱。

## 3. 悬疑惊悚

### 叙事发动机

使用未知来源、证据链、有限视角、迫近危险和错误假设修正推动故事。

悬疑不是把信息全部藏起来。必须给观众公平线索。

### 常见人物目标

- 找到失踪物或人物。
- 确认声音、影像或痕迹来源。
- 在时限内离开或阻止行动。
- 证明某个说法不成立。
- 判断谁掌握关键事实。

### 可见证据

优先使用：

- 门锁、脚印、湿痕、时间戳。
- 监控盲区。
- 物品位置变化。
- 不匹配的声音。
- 缺失文件。
- 镜中、玻璃或阴影中的局部信息。

不要只写“阴森”“诡异”。

### 钩子

- 视觉异常。
- 逼近危险。
- 戏剧性反讽。
- 规则破坏。
- 未完成行动。

### 延期规则

每次延期至少执行一项：

- 新增可靠线索。
- 排除一个可能。
- 提高危险。
- 暴露证词矛盾。
- 改变人物知识状态。

不要用无关闪回或突然昏迷拖延答案。

### 节奏

- 在快节奏行动与短暂停顿之间形成差异。
- 为线索读取保留时间。
- 声音来源可延迟揭示，但要记录 Hook Contract。
- 惊吓后必须产生行动或信息结果。

### 台词

减少解释性对白。让角色围绕证据说话：

```text
“门从里面锁着。”
“那这串湿脚印是谁的？”
```

避免：

```text
“这个房间让我感到一种无法言说的危险。”
```

### 声音

优先设计：

- 可定位或故意不可定位的脚步。
- 门锁、呼吸、震动、提示音。
- Room Tone 状态变化。
- 合法 J-cut 声音预告。

若声音是关键线索，记录来源状态和兑现。

### Coverage

优先：

- 主观视线目标。
- 证据 Insert。
- 空间建立。
- 门口、出口和遮挡。
- 发现线索后的行动反应。

### 生成风险

- 镜面、反射、阴影。
- 黑暗中的人物一致性。
- 小物体位置连续。
- 复杂物理痕迹。

降低反射复杂度，固定机位，使用独立 Insert 或合成。

### 禁止推断

- 超自然力量，除非世界规则已建立。
- 无因出现的凶手或危险。
- 用精神疾病作为随意反转。
- 只为惊吓新增暴力或血腥。

## 4. 家庭伦理

### 叙事发动机

使用责任、照护、财产、代际边界、公开与私下承诺之间的冲突推动故事。

不要默认使用出轨、真假血缘或恶婆婆模板。

### 常见人物目标

- 分配照护责任。
- 保护家庭成员。
- 确认财产或居住安排。
- 公开隐瞒事项。
- 拒绝不公平义务。
- 保存关系同时建立边界。

### 可见证据

优先使用：

- 房产证、医疗单据、转账、钥匙。
- 餐桌座位。
- 行李、药盒、看护记录。
- 家庭群消息。
- 谁签字、谁付款、谁离场。

### 钩子与爽点

钩子：

- 家庭承诺与现实记录冲突。
- 关键签字或财产安排即将发生。
- 某人公开拒绝责任。

爽点：

```text
所有人要求她独自承担费用。
她把连续六个月的转账记录投到电视上。
一直沉默的家人把付款单推回责任人面前。
```

### 节奏

- 不让所有人物轮流陈述立场。
- 用一个具体决定推动每轮冲突。
- 让长辈、孩子或旁观者反应服务行动，不只煽情。
- 在情绪爆发后落到签字、付款、离场或照护安排。

### 台词

让台词体现关系和条件：

```text
“你说我没管过家。账单在这里。”
“房子可以卖，先把护理费写进协议。”
```

避免道德宣判式长句：

```text
“你们这些人从来不知道亲情到底意味着什么。”
```

### Coverage

优先：

- 餐桌或客厅关系镜头。
- 单据 Insert。
- 谁控制文件或钥匙。
- 关键家庭成员的选择反应。

### 声音

- 保留沉默、餐具、电视底声等家庭空间声音。
- 不用持续煽情音乐压住真实停顿。
- 重叠争吵中保证关键条件可懂。

### 生成风险

- 多代同堂多人场景。
- 年龄和身份一致性。
- 密集重叠对白。
- 餐桌道具连续。

减少同镜运动主体，拆反应，记录道具状态。

### 禁止推断

- 未说明的血缘秘密。
- 未说明的重病、遗嘱或死亡。
- 把某一年龄或家庭角色自动写成反派。
- 用儿童承担不合适的成人冲突信息。

## 5. 职场商战

### 叙事发动机

使用权限、合同、资源、流程、投票、证据和交易条件推动故事。

所有权力变化必须有组织规则或行动结果支持。

### 常见人物目标

- 证明项目归属。
- 争取预算、职位或合作。
- 阻止错误签约。
- 揭示违规流程。
- 保护客户、数据或商业秘密。
- 改变会议决策。

### 可见证据

优先使用：

- 合同版本号。
- 邮件时间戳。
- 权限页面。
- 投票结果。
- 印章。
- 会议纪要。
- 原型或数据看板。
- 转账和审批记录。

确保文字可读且不伪造真实品牌或敏感数据。

### 钩子

- 系统权限异常。
- 关键文件版本冲突。
- 会议即将表决。
- 对手公开指控。
- 证据在最后时限前到达。

### 爽点

```text
对手展示修改后的方案，声称原创。
主角打开版本历史，最早提交记录显示自己的账号和日期。
客户把签字笔从对手面前移到主角面前。
```

### 节奏

- 每轮会议对话必须改变条件、证据或票数。
- 避免人物轮流朗读商业背景。
- 用屏幕、文件和动作压缩说明。
- 爽点后明确权限、合作或决策变化。

### 台词

使用专业但可懂的短句：

```text
“你展示的是第三版。原始提交记录在这里。”
“没有审计签字，这份付款单不能执行。”
```

避免堆专业词制造权威。

### Coverage

优先：

- 会议空间关系。
- 文件与屏幕 Insert。
- 投票、签字或权限验证动作。
- 决策者反应。
- 可用于压缩长台词的听者和资料画面。

### 声音

- 键盘、投影、打印、会议静默可成为节奏点。
- 远程会议音频要明确来源和延迟。
- 商业秘密对白注意权利和隐私。

### 生成风险

- 大量准确文字。
- 多人会议。
- 屏幕与手部交互。
- 品牌标识。

优先后期合成屏幕，使用匿名化界面，拆独立 Insert。

### 禁止推断

- 无程序即时开除。
- 无来源的董事身份。
- 夸张金额或公司估值。
- 用真实公司、商标或个人影射违法行为。

## 6. 喜剧误会

### 叙事发动机

使用人物对同一动作、物体、称呼或规则作出不同解释，经过重复升级或视角切换完成笑点。

误会必须有可见原因，不能只靠人物不听解释。

### 常见人物目标

- 隐瞒一个无害事实。
- 完成任务而不暴露身份。
- 纠正错误对象。
- 避免尴尬。
- 争取发言或解释机会。

### 可见证据

优先使用：

- 相似包装。
- 错误座位。
- 同名联系人。
- 被遮挡的标识。
- 一半被听见的对话。
- 道具交换。

### 钩子

- 观众先看见误会来源。
- 人物即将把错误物品交给错误对象。
- 一句称呼被另一个人听见。
- 规则建立后立刻被意外打破。

### 笑点与爽点

使用：

```text
建立规则
→ 第一次偏差
→ 人物补救
→ 补救造成更大偏差
→ 具体揭示
→ 反应
```

不要无限重复同一误会。

### 节奏

- 给观众提前看见信息的时间。
- 在动作结果后保留短反应。
- 重复必须升级变量，不只换人物再来一次。
- 解释应在最有结果的节点发生，不要无故拖延。

### 台词

让台词具有双重指向，但两种理解都必须由上下文支持：

```text
A：“你带来的那个，先放桌下。”
B 以为说的是礼物，实际 A 指的是维修工具。
```

避免角色为了维持误会突然不说正常人会说的话。

### Coverage

优先：

- 观众知道真相的 Insert。
- 误会人物的视线目标。
- 同框关系镜头。
- 动作结果和反应。
- 揭示时的双人或群体状态。

### 声音

- 停顿、误听、门外半句和物体声可构成笑点。
- 不用廉价笑声音轨替代笑点结构。
- 关键双关要验证字幕是否破坏或帮助理解。

### 生成风险

- 精准物体交换。
- 多人同步节奏。
- 复杂连续动作。
- 表情时机。

拆成短动作，使用反应镜头和独立道具 Insert。

### 禁止推断

- 角色无故降智。
- 用口音、残障、外貌或身份群体作为默认笑点。
- 通过长期隐瞒严重事实维持轻喜剧。
- 用解释旁白告诉观众哪里好笑。

## 7. 古装权谋

### 叙事发动机

使用身份等级、礼制位置、诏令、印信、密信、证词和公开程序推动权力变化。

必须建立世界中的规则，再让人物利用或违反规则。

### 常见人物目标

- 取得召见或发言资格。
- 保护身份或秘密。
- 证明诏令、印信或证词真伪。
- 改变联盟。
- 阻止命令执行。
- 在礼制范围内公开拒绝。

### 可见证据

优先使用：

- 印信、腰牌、诏书、座次、帘幕。
- 跪、起、让位、拦路等制度化动作。
- 门禁、侍卫响应、传令顺序。
- 墨迹、封蜡、纸张和持有者。

不要用现代合同和职场语言直接替换。

### 钩子

- 诏令与印信不匹配。
- 不应出现的人坐在特定位置。
- 封存密信已被打开。
- 传令内容在公开场合被质疑。
- 规则被高风险地打破。

### 爽点

```text
对手命侍卫拿人。
主角举起可验证腰牌。
侍卫停步，转身向持牌者行礼。
```

前提是腰牌权限已在世界规则中建立。

### 节奏

- 先建立礼法和风险，再展示越规行动。
- 权谋对话每轮应改变信息、联盟或执行状态。
- 重要诏令和证据给足读取或口述时间。
- 不连续堆“还有一道圣旨”。

### 台词

区分身份与场合，但保持可说：

```text
“此印不在尚书台备案。谁准你传令？”
```

避免现代网络爽文口号或过度文言导致演员无法自然表达。

### Coverage

优先：

- 座次与等级空间。
- 腰牌、印信、密信 Insert。
- 发令者、执行者和被执行者的关系镜头。
- 侍卫停止、让路、转向等权力结果。

### 声音

- 脚步、衣料、木门、传令声和环境静默可建立秩序。
- 不随意使用不符合世界的现代提示音。
- 群体口号和宣令保证主信息清楚。

### 生成风险

- 服装、发饰、群演一致性。
- 长袖和手部。
- 复杂宫殿空间。
- 大量人群动作。
- 文字和印章准确性。

减少同镜群演动作，锁定服装参考，证据用独立 Insert 或合成。

### 禁止推断

- 未建立的官职权限。
- 无来源的密旨、血统或皇族身份。
- 不符合项目世界规则的礼制。
- 用现代法律程序冒充古代制度。

## 8. 奇幻科幻

### 叙事发动机

使用明确世界规则、能力限制、技术操作和代价推动故事。

任何能力或装置都必须有输入、输出、限制和可见结果。

### 常见人物目标

- 启动、关闭或修复装置。
- 验证能力规则。
- 在资源耗尽前完成行动。
- 隐藏或公开非正常身份。
- 阻止规则失控。
- 选择是否承担能力代价。

### 可见证据

优先使用：

- 能量或资源计量。
- 装置状态灯。
- 重复实验结果。
- 身体或环境的具体变化。
- 规则触发前后的对象状态。
- 日志、地图和计时。

不要只写“强大能量”“时空扭曲感”。

### 钩子

- 规则被打破。
- 装置出现不可能读数。
- 能力产生意外代价。
- 已删除的人或数据重新出现。
- 倒计时与资源耗尽。

### 爽点

爽点来自人物理解并利用规则：

```text
装置只能识别活体指纹。
主角把自己的手按上识别区，系统打开被封锁的实验室。
追兵的复制手套显示验证失败。
```

不要临时新增万能能力解决冲突。

### 节奏

- 先用最少画面建立规则。
- 通过行动验证规则，而不是长篇讲解。
- 规则例外必须提前有线索。
- 特效段落后给出清楚结果和代价。

### 台词

使用操作目标和限制：

```text
“电量只够开一次门。”
“门开后，系统会锁死。”
```

避免伪科学长句和无功能术语。

### Coverage

优先：

- 装置状态 Insert。
- 操作者与结果同一因果链。
- 能力前后状态。
- 资源计量。
- 角色承受代价的反应。

### 声音

- 为装置状态建立一致声音词典。
- 同一警报不要在不同场景表示相反状态。
- 能力启动、失败和耗尽使用可区分声音。
- 关键规则仍需视觉冗余。

### 生成风险

- 复杂 VFX。
- 身体变形。
- 多主体特效交互。
- 精确界面文字。
- 物理规则连续。

拆分特效阶段，使用固定摄影机、Clean Plate、合成和独立界面 Insert。

### 禁止推断

- 万能能力。
- 临时规则例外。
- 未说明的复活、时间旅行或读心。
- 用视觉炫技掩盖规则矛盾。

## 跨题材台词规则

### 每轮对白必须有功能

标记每一轮：

```text
request
refusal
condition
evidence
threat
choice
reveal
misdirection
correction
decision
```

连续两轮功能相同且没有新信息时，考虑压缩。

### 控制信息单位

一句台词中的独立信息点过多时：

1. 保留当前行动必须的信息。
2. 把可见内容交给画面。
3. 把次要背景延后。
4. 不因压缩改变事实。

### 保持角色语言差异

通过以下维度区分：

- 句长。
- 直接程度。
- 称呼。
- 专业词。
- 是否先给结论。
- 是否回避。
- 是否打断。

不要只靠口头禅区分角色。

### 爽点台词

爽点台词应落在证据或行动之后：

```text
她把授权页投到屏幕上：“现在，重新投票。”
```

不要让角色提前口头宣布尚未发生的胜利。

## 跨题材生成风险

### 高风险共同项

- 三人以上同时进行独立动作。
- 多人口型同步。
- 手部交换小道具。
- 手机、合同、屏幕的准确文字。
- 镜面和反射。
- 群体同步反应。
- 长袖、复杂服装和快速肢体接触。
- 复杂特效与摄影运动同时发生。

### 通用降级顺序

1. 减少同时运动主体。
2. 删除不承担 Beat 的次要动作。
3. 把文字、屏幕和小道具拆成 Insert。
4. 固定摄影机。
5. 把口型改为单人或画外。
6. 拆分动作阶段。
7. 使用参考图、首尾帧或 Clean Plate。
8. 改用真人补拍、后期合成或静态插入。

不要通过删除关键证据降低风险。

## 正反例

### 题材选择

```text
坏：
项目同时标注八种主类型，并为每种都添加一套钩子。

好：
主类型为职场商战；悬疑只用于合同版本线索的延期揭示。
```

### 爽点

```text
坏：
她气场全开，所有人震惊。

好：
她把投票结果投到屏幕上；主持人收回对手的发言牌，把它放到她面前。
```

```text
坏：
为了爽，让对手突然承认所有错误。

好：
对手先否认；录音播放出其说出的文件编号后，他伸手关播放器，被保安拦住。
```

### 节奏

```text
坏：
每两秒必须有一次反转。

好：
每个段落至少形成一次状态变化；反转只在新证据足以改变原判断时使用。
```

```text
坏：
连续五轮争吵，双方只重复“我没错”。

好：
每轮分别改变证据、条件、支持者和最终行动。
```

### 台词

```text
坏：
“你以为我还是从前那个任你欺负、没有能力也没有背景的我吗？”

好：
她把退回的合同放到桌上：“这次，我不签。”
```

### 都市情感

```text
坏：
用旁白解释她已经放下。

好：
她归还钥匙，拖走自己的行李，门在两人之间关上。
```

### 逆袭

```text
坏：
主角突然亮出未建立的首富身份。

好：
用原稿已有的系统权限、原始文件或投票结果完成反转。
```

### 悬疑

```text
坏：
黑暗里随机出现人影，只为吓观众。

好：
先建立房间只有一个入口；门仍锁着，地面却出现向内延伸的湿脚印。
```

### 家庭伦理

```text
坏：
所有冲突都靠哭喊和道德指责。

好：
把照护、付款和居住安排落到记录、钥匙和签字。
```

### 职场商战

```text
坏：
堆砌“资本、估值、并购、对赌”却没有动作结果。

好：
会议在一个具体表决、权限变更或签字动作中改变状态。
```

### 喜剧

```text
坏：
角色只要听完一句话就能解除误会，却无故拒绝听。

好：
误会来自被遮挡标签和同名联系人；每次补救改变物体位置并升级结果。
```

### 古装

```text
坏：
人物拿出一块未建立权限的腰牌，所有人立即下跪。

好：
先建立腰牌可调动哪一组侍卫；兑现时只让对应侍卫停止执行命令。
```

### 奇幻科幻

```text
坏：
危急时刻人物突然获得新能力。

好：
人物利用前段已经验证过的规则，并承担已建立的资源代价。
```

## 硬失败

出现以下任一项，必须退回：

- 题材 Profile 被当作新增剧情授权。
- 爽点没有前置压制、人物行动、证据或结果。
- 为制造爽点让对手无因降智。
- 新增身份、血缘、犯罪、疾病、死亡、财富或能力但没有授权。
- 连续多个 Beat 没有任何状态变化，却被标记为快节奏。
- 台词压缩改变人物立场或事实。
- 反转违反已建立因果或知识状态。
- 钩子没有 Hook Contract 或合法兑现。
- 关键证据无法在画面或声音中读取。
- Profile 未记录版本和来源状态。
- 多题材规则冲突但没有主 Profile。
- 题材套路触发违法、危险、歧视或平台敏感内容而未进入合规审查。
- 为降低生成风险删除关键剧情证据。

## 软警告

以下情况允许保留，但必须评估：

- 主 Profile 与两个辅助 Profile 同时活跃。
- 连续两个段落使用相同反转方式。
- 爽点主要依赖旁观者反应。
- 主角连续获胜，没有成本、阻碍或选择。
- 对手承担全部解释性台词。
- 每段都以未完成行动结尾，造成疲劳。
- 题材语体过强，演员难以自然说出。
- 关键道具重复承担过多证据功能。
- 喜剧误会延期过长。
- 悬疑线索依赖极小文字。
- 古装和科幻项目的世界规则只存在于备注栏。
- 题材 Profile 的参数未经当前平台和受众测试。
