## 1. 使用目的

使用本手册完成任务分流、权限锁定和阶段路由。

始终先确定“允许改到哪里”，再分析剧情、设计镜头或增强钩子。

默认执行：

```text
忠实转换
+ 最小必要补全
+ 原子分镜
+ 不新增剧情事实
```

不得把“优化分镜”自动解释为“重写剧情”。不得把“Seedance 可生成”自动解释为“必须删掉剧情信息”。

## 2. 先建立 JobSpec

在处理正文前创建任务配置。至少记录：

```yaml
job_id:
schema_version:
input_type:
production_mode: live | seedance | hybrid | undecided
revision_permission: faithful | visual | pacing | story
platform:
aspect_ratio:
target_duration_seconds:
episode_duration_seconds:
audience:
regions:
language:
genre:
budget_level:
budget_limit:
audio_generation:
content_rating:
seedance_model_profile_id:
atomic_shot_policy: default | long_take_exception
```

遵守以下规则：

1. 为每项用户明确要求标记 `explicit`。
2. 为每项推断值标记 `inferred`。
3. 为每项默认值标记 `defaulted`。
4. 不把推断值伪装成用户要求。
5. 不确定制作方式时使用 `undecided`，不要抢先锁定 Seedance。
6. 未确认模型版本时引用模型注册表中的候选档案，不自行编造能力。

## 3. 识别输入模式

### 3.1 模式 A：完整剧本

在输入具有场景标题、动作、人物、对白或连续剧情时使用。

执行：

1. 保存不可覆盖的原稿。
2. 建立稳定来源范围。
3. 拆分 Scene。
4. 建立事实台账和 Story Bible。
5. 拆分 Beat。
6. 完成调度、Coverage、Shot 和生成任务。

不得按段落或句号机械生成镜头。

### 3.2 模式 B：故事梗概

在输入只说明主要事件、人物关系或结局时使用。

执行：

1. 保留梗概中的明确事实。
2. 标记缺失的地点、时间、行动方式和空间关系。
3. 仅补足“让现有事件可被拍摄”所需的最小动作。
4. 将任何非必要扩写列为建议，不直接写入正式剧情。

把“她反击老板”补成“她把录音笔放到会议桌上”属于视觉化候选；把老板改成她的亲生父亲属于新增剧情事实。

### 3.3 模式 C：对白稿

在输入主要由人物对白组成，缺少动作和空间信息时使用。

执行：

1. 识别说话者、说话顺序和对白目的。
2. 推导最小可执行站位与反应。
3. 区分同期对白、画外对白、旁白和可后期配音对白。
4. 为关键对白绑定可见动作或反应。
5. 标记无法从原文确认的场景和人物状态。

不得为每句对白机械分配一个镜头。

### 3.4 模式 D：已有分镜

在输入已经包含镜头编号、景别、机位、动作或时长时使用。

先诊断，再修改。建立：

```text
原镜头
→ 问题
→ 修复权限
→ 修改后镜头
→ 差异
```

默认进入“仅修复”，不得自动进入创意优化。

### 3.5 模式 E：Seedance 提示词

在输入主要是视频生成提示词时使用。

执行：

1. 反向还原主体、场景、动作、摄影机、声音和时间顺序。
2. 识别提示词内隐含的 Shot。
3. 检查是否存在多景别、隐含剪辑、时空跳跃或动作过载。
4. 生成缺失信息清单。
5. 在来源不足时将还原内容标记为 `inferred`。

不得把提示词中的风格词当作原剧情事实。

### 3.6 混合输入

当剧本、已有分镜和提示词同时存在时，按以下优先级确定事实来源：

```text
用户锁定事实
→ 最新授权修订稿
→ 原始剧本
→ 已有分镜
→ 生成提示词
```

遇到冲突时不要静默选择。记录冲突，保留来源版本，并路由到“来源冲突确认”。

## 4. 锁定四级修改权限

### 4.1 `faithful`：忠实转换

允许：

- 拆 Scene、Beat 和 Shot。
- 把心理信息转为不改变事实的可见或可听证据。
- 补足动作起点、终点、站位和必要承接。
- 调整语序以适配结构化输出。
- 删除纯作者解释，但保留它承载的剧情信息。

禁止：

- 新增人物关系、秘密、犯罪、死亡、怀孕、血缘或背叛。
- 改变事件结果。
- 改变人物核心动机。
- 调整关键事件先后顺序。
- 为增强钩子制造原文不存在的危险。

将此模式设为默认模式。

### 4.2 `visual`：视觉增强

在 `faithful` 基础上允许：

- 选择更清楚的动作、道具、构图和声音表达。
- 调整人物距离和空间层次。
- 增加不改变剧情事实的环境反应。
- 为既有信息安排插入镜头或反应镜头。
- 用视觉反差强化原有冲突。

仍然禁止新增剧情事实或改变因果。

### 4.3 `pacing`：节奏优化

在 `visual` 基础上允许：

- 调整镜头长度。
- 拆分、合并或局部重排镜头。
- 前置原文已有的结果、证据或异常。
- 延迟揭示原文已有的信息。
- 调整反应镜头和段尾悬念位置。
- 压缩重复表达。

不得改变核心因果、事件结局或人物身份。

任何重排都必须保留原始时间顺序记录，并输出差异。

### 4.4 `story`：剧情优化

仅在用户明确授权时启用。

允许在授权范围内：

- 新增钩子。
- 增加冲突或误导。
- 改写事件顺序。
- 新增或删除剧情信息。
- 改变人物选择或结果。

执行前记录：

```yaml
authorization_id:
authorized_scope:
locked_elements:
content_limits:
approved_by:
approved_at:
```

不得把模糊的“更爽一点”解释为无限剧情改写。只在用户授权的题材、尺度、人物和段落范围内修改。

### 4.5 权限升级

只有满足以下条件之一时升级权限：

- 用户直接指定更高权限。
- 用户批准一份明确列出变化的建议。
- 当前任务无法在原权限内完成，且用户确认扩展范围。

不得因技术困难自动升级权限。

权限降低可以随时执行；降低后立即停止尚未应用的越权修改。

## 5. 选择制作模式

### 5.1 真人拍摄 `live`

优先选择真人拍摄：

- 长对白依赖精确表演。
- 多人需要精确口型和反应。
- 存在精细手部交互。
- 用户已有演员、场地和授权。
- AI 重试成本高于拍摄成本。

### 5.2 Seedance `seedance`

优先选择 Seedance：

- 场景难以取得或实拍风险高。
- 动作短、主体清楚、状态可锁定。
- 幻想环境或大规模视觉变化占主导。
- 参考素材足以维持身份和场景一致性。

不要因为项目总体使用 Seedance，就把每个 Shot 都强制设为生成。

### 5.3 混合制作 `hybrid`

在真人表演与生成环境、合成效果或局部补拍结合时使用。

额外要求：

- 记录 Clean Plate。
- 记录跟踪和遮罩需求。
- 对齐帧率、快门、分辨率和色彩空间。
- 对齐光向、景深和颗粒。
- 明确同期声与后期声音交接。

### 5.4 逐镜决策

为每个 Shot 记录：

```yaml
primary_method:
fallback_method:
decision_reasons:
switch_conditions:
required_handoff_assets:
```

当 Seedance 连续三轮仍有阻断失败，或预期重试成本超过替代方案时，退回制作方式决策，不要继续盲目重试。

## 6. 执行统一工作流

按以下顺序执行：

1. 冻结原始输入和版本。
2. 建立 JobSpec。
3. 锁定修改权限。
4. 建立来源清单和事实台账。
5. 解析 Scene、人物、地点、道具和对白。
6. 建立 Story Bible。
7. 拆分 Beat。
8. 检查钩子和短剧结构。
9. 把抽象信息转为可见或可听证据。
10. 读取平台与画幅约束。
11. 设计人物调度和空间关系。
12. 标记声音与剪辑意图。
13. 建立 Coverage Need 和临时 Shot Intent。
14. 规划 Camera Setup。
15. 冻结原子 Shot。
16. 建立事件时间轴、声音和时长。
17. 写入连续性状态。
18. 逐镜选择制作方式。
19. 编译 Generation Task。
20. 执行质量门禁。
21. 将错误退回责任阶段。
22. 输出分镜、任务表、差异和风险报告。

不得跳过来源锁定直接写镜头。

## 7. 执行已有分镜优化工作流

### 7.1 仅修复

修复：

- 不可见描述。
- 单镜多景别。
- 隐含剪辑。
- 时长不可执行。
- 连续性冲突。
- Shot 与提示词不一致。

默认保持镜头顺序。只有当前顺序造成不可拍、不可剪或连续性阻断时，执行最小局部重排并记录理由。

### 7.2 生成适配

在仅修复基础上：

- 拆分 Generation Task。
- 绑定参考素材 ID。
- 设置首尾帧目标。
- 降低主体、动作和摄影运动负载。
- 设置 Attempt 验收和降级方案。

不得为了模型方便删除关键 Beat。

### 7.3 创意优化

仅在 `pacing` 或 `story` 权限下执行。

区分：

```text
已有信息重排
呈现方式变化
授权新增剧情
```

不得把三者混成“优化”。

## 8. 管理默认值与缺失信息

### 8.1 可安全默认

仅对不改变剧情的技术字段使用默认值，例如：

- 未指定时使用原子 Shot 策略。
- 未指定修改权限时使用 `faithful`。
- 未指定制作方式时使用 `undecided` 并逐镜评估。
- 未指定景别时根据 Beat 和平台提出候选，不直接宣称为原稿要求。

### 8.2 必须确认或阻断

遇到以下信息缺失时，不得擅自补全为事实：

- 人物身份或关系。
- 关键证据内容。
- 事件结果。
- 死亡、生存、怀孕、血缘、犯罪等高影响事实。
- 时间顺序会改变因果的节点。
- 外部素材权利。
- 未成年人、肖像和声音授权。

可以继续完成不依赖该事实的部分，并把相关对象标记为 `blocked_by_unknown`。

### 8.3 使用假设

必须使用假设时：

1. 标记 `assumption_id`。
2. 写明依据。
3. 写明受影响对象。
4. 设置可替换的默认方案。
5. 不把假设写入锁定事实台账。

## 9. 输出差异与授权记录

每次转换至少输出以下差异类别：

```text
保留事实
删除或压缩
视觉化
顺序变化
推断补全
授权新增
冲突或待确认
```

为每项变化记录：

```yaml
change_id:
artifact_id:
field_path:
before:
after:
transformation_type:
source_refs:
fact_ids:
authorization_id:
reason:
```

任何 `authorized_addition` 都必须具有 `authorization_id`。

## 10. 正反例

### 10.1 权限判断正例

用户要求：

```text
忠实转成分镜，不要改剧情，但开头尽量抓人。
```

正确处理：

```text
revision_permission: faithful
```

从原文已有冲突、物证或结果中选择首屏画面；将其他钩子方案列为未采用建议。

错误处理：

```text
凭空增加女主婚礼当天发现丈夫出轨。
```

### 10.2 梗概补全正例

原文：

```text
她拿证据反击经理。
```

可接受：

```text
她把手里的证据放到会议桌中央。
```

只在原文已经说明证据是录音笔、文件、照片等具体媒介时，才能保留该媒介并标记为 `visualized`。若媒介、日期、金额、内容或持有人不是原文事实，必须标记为 `inferred` 并允许替换；一旦该细节进入证据链、因果、道具连续性或后续兑现，就必须取得授权，不能继续当作自由视觉化。

不可接受：

```text
录音里传出经理收受五十万元的声音。
```

该金额和犯罪事实不在原文中。

### 10.3 `visualized`、`inferred` 与新增事实

- `visualized`：只改变同一事实的可见表达，不增加新的证据媒介、属性、关系或因果。
- `inferred`：为可拍性暂定、可撤销且不进入锁定事实的细节。
- `authorized_addition`：新增会影响因果、证据链、人物判断、连续性或后续兑现的事实。

当无法确定分类时，默认使用 `inferred`，不要用 `visualized` 掩盖新增信息。

### 10.4 制作模式正例

多人长对白使用真人主拍；城市坍塌外景使用 Seedance；将演员绿幕表演与生成外景合成。把每镜的方式和切换条件分别记录。

反例：

```text
因为项目写了 Seedance，所以所有对白镜头都强制 AI 生成。
```

## 11. 错误路由

| 错误 | 严重级别 | 返回阶段 | 处理 |
|---|---|---|---|
| 输入类型无法确定 | warning | JobSpec | 保留多个候选入口并标记待确认 |
| 原稿版本冲突 | blocker | 来源锁定 | 冻结各版本，要求指定主版本 |
| 未确认修改权限 | error | JobSpec | 使用 `faithful`，禁止创意改写 |
| 未授权新增剧情 | blocker | 产生新增内容的阶段 | 删除新增或取得授权 |
| 缺少关键剧情事实 | error | 剧本分析 | 标记未知，不自行编造 |
| 钩子需要改剧情 | warning | 结构编辑 | 作为可选建议，不写入正式分镜 |
| 制作方式不适配 | error | 制作决策 | 比较真人、生成、混合并重选 |
| 技术限制导致剧情丢失 | blocker | Shot/Task 设计 | 改用拆镜、Coverage 或替代制作 |
| 已有分镜无法最小修复 | error | 权限确认 | 提交必要重排范围，不越权执行 |
| 权利或合规状态未知 | blocker | 权利审核 | 阻断相关素材进入最终 Clip |

## 12. 完成门禁

只有同时满足以下条件才允许进入下游镜头设计：

- 已建立 JobSpec。
- 已识别输入模式。
- 已锁定修改权限。
- 已冻结主原稿和版本。
- 已声明制作模式或保留逐镜决策。
- 已记录默认值、假设和未知项。
- 已建立授权边界。

只有同时满足以下条件才允许交付：

- 没有未授权剧情新增。
- 所有剧情变化均有差异记录。
- 所有阻断错误已解决或明确标记为阻塞。
- 生成适配没有删除关键 Beat。
- 正式输出与建议区分开。
- 输出明确说明哪些内容来自原文、哪些属于视觉化、推断或授权新增。
