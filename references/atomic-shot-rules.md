# 原子分镜规则

把剧情节拍和 Provisional Shot Intent 冻结为可拍、可剪、可生成的 Shot。默认采用“一个主导景别、一个连续时空、一个主要行动单元、一个主要叙事目的”的原子模式；不要把该生产策略误写成普遍电影理论。

## 术语边界

始终分开以下对象：

| 对象 | 含义 |
|---|---|
| Beat | 一次行动、信息或局势变化 |
| Provisional Shot Intent | 尚未冻结摄影参数的镜头意图 |
| Shot | 成片中两个剪辑点之间的连续画面 |
| Camera Setup | 一套机位、镜头、灯光和空间配置 |
| Take | 真人拍摄对某个 Shot 的一次执行 |
| Generation Task | 一次结构化 AI 生成任务 |
| Media Asset | Take、Attempt 或合成产生的素材 |
| Timeline Clip | 时间线上对素材某一区间的一次使用 |

不要假定：

- 一个 Beat 必然等于一个 Shot。
- 一个 Shot 必然等于一个 Generation Task。
- 一个生成结果可以整段直接作为最终 Shot。
- 一个 Camera Setup 只能服务一个 Shot。

## 原子镜头契约

默认原子 Shot 满足：

```text
一个连续空间
+ 一个连续时间
+ 一个主导景别
+ 一个主要注意力中心
+ 一个主要行动单元
+ 一个主要摄影意图
+ 一个开始状态
+ 一个结束状态
+ 一个主要叙事目的
```

“一个主要行动单元”允许包含不可分割的准备、执行和即时结果。例如：

```text
她举起钥匙
→ 放到桌上
→ 松手
```

这是一个完整行动单元，不必机械拆成三个镜头。

“一个主要叙事目的”允许关系镜头中同时出现对方的即时反应，只要反应服务同一目的，没有转移为新的独立信息。

### 必填问题

冻结 Shot 前回答：

1. 观众主要看谁或什么。
2. 画面从什么状态开始。
3. 发生哪个主要行动单元。
4. 画面以什么可见状态结束。
5. 镜头传递哪项主要信息。
6. 它满足哪个 Coverage Need。
7. 在哪里切出，切镜动机是什么。
8. 它是否能由所选制作方式执行。

## 主导景别

### 定义

`dominant_shot_size` 是镜头大部分有效时长中主导观众读取方式的景别，不要求像素级构图完全静止。

推荐使用稳定枚举：

```text
extreme_wide
wide
full
medium_wide
medium
medium_close
close_up
extreme_close_up
insert
over_shoulder
two_shot
group_shot
pov
other
```

避免同时写“中近景/近景/特写，看情况”。

### 同级构图漂移

默认允许：

- 人物呼吸造成的轻微画面变化。
- 人物在原 Mark 附近小幅前后移动。
- 小幅推、拉、摇或移用于保持构图。
- 人物转身、坐下或起身后仍维持同一主导读取尺度。
- 对焦从同一景别内的前景人物转到后景人物。

使用 `allowed_composition_drift` 明确范围：

```yaml
allowed_composition_drift: same_size_minor
```

只使用 Schema 枚举：`none`、`same_size_minor`、`adjacent_size_continuous`、`declared_long_take_path`。若需要记录比例阈值或重新构图原因，将其写入 `start_composition`、`end_composition`、`camera_motion.path` 或长镜头例外说明，不要把该字段改成对象。

### 跨级变化

以下通常视为主导景别跨级：

- 全景推进到近景。
- 双人中景移动为单人特写。
- 人物从全身走到只剩脸部。
- 画面注意力由环境建立转为小道具细节。

默认拆镜。只有启用连续长镜头例外时保留。

### 关系镜头

双人或多人镜头可以是原子 Shot。不要把“一个主要主体”误解为画面只能出现一个人。

允许：

```text
双人中景中，A 把合同推给 B，B 不接。
```

主要注意力中心是“合同交接失败的关系行动”，两人共同构成一个单元。

不允许：

```text
双人中景中 A 推合同，镜头随后转向门口的 C，再特写桌下录音笔。
```

这里出现三个独立注意力中心，应拆镜。

## 动作和主体负载

### 主要主体

主要主体是观众必须读取其行动或状态的人、物或群体。背景人物不自动算主要主体。

允许多个主体同时存在，但要满足：

- 主次关系清楚。
- 同时运动数量可控。
- 动作具有共同因果。
- 不要求观众同时读取多个独立关键信息。

### 主要行动单元

用以下测试判断是否仍是一个单元：

> 删除其中一个动作后，当前结果是否仍成立？

若两个动作可以各自形成独立结果，通常拆镜。

```text
可保留：
她打开信封，抽出照片。

应拆：
她打开信封，读完照片背面的字，又冲出房间拦车。
```

### 动作负载

评估：

- 同时运动的主要主体数量。
- 动作链节点数量。
- 手部和小道具精度。
- 接触、遮挡和物理互动。
- 台词、表情与摄影运动是否同时争夺注意力。
- 在规定时长内是否能完成并留出反应。

动作负载高不必立即失败，但要拆镜、简化或标记高风险。

### 开始和结束状态

使用可拍状态，不写抽象结果：

```text
坏：
start_state: 她仍然被压制
end_state: 她夺回主动权

好：
start_state: 合同和遥控器都在经理面前
end_state: 合同停在她手边，她握着遥控器，屏幕显示转账记录
```

## 拆镜触发器

出现以下变化时，默认创建新 Shot：

### 1. 主导景别跨级

从环境读取切换为人物表情或小道具细节。

### 2. Camera Setup 或空间方向改变

切到反打、过肩、俯拍、门外或桌下。

### 3. 注意力中心改变

从说话者切到听者的独立反应，从人物切到证据。

### 4. 新信息结果出现

动作产生可独立读取的后果，例如屏幕亮起、门后出现人、文件签名被揭示。

### 5. 时空改变

地点、时间、主观层级或现实状态改变。

### 6. 新动作链开始

前一行动已经结束，人物开始具有新目标的下一行动。

### 7. 需要独立反应

反应本身改变关系、暴露秘密或成为剪辑节奏点。

### 8. 生成或实拍负载超限

镜头在理论上连续，但当前制作方式无法稳定完成。

### 9. 需要编辑控制

关键动作、对白或钩子需要单独调节时长、顺序或揭示。

不要仅按句号、台词轮次或固定秒数拆镜。

## 允许保留在同一镜头的情况

满足全部条件时，可以保留：

- 时空连续。
- 主导景别不跨级。
- 没有隐含剪辑。
- 行动和反应属于同一因果单元。
- 观众注意力不需要跳到新信息中心。
- 调度、时长和制作方式可执行。

典型情况：

### 动作与即时结果

```text
她按下播放键，录音从扬声器响起。
```

### 边听边反应

```text
双人中景中 A 说出解约条件，B 的手从合同上移开。
```

### 连续交接

```text
A 把钥匙递出，B 没有接，钥匙停在两人之间。
```

### 同景别内重新构图

```text
人物从椅背后向前半步，摄影机小幅后移保持中景。
```

### 对焦转移

同一构图内从前景证据转焦到后景人物，可以保留；若证据和反应都承担独立重大信息，仍建议拆镜。

## 摄影机运动

### 默认一个主要运动

每镜优先使用：

```text
固定
或
一个主要推 / 拉 / 摇 / 移 / 跟 / 升降 / 环绕
```

允许为保持主体构图做小幅修正，不把每次修正写成新的主要运镜。

### 运动必须有叙事理由

```text
坏：镜头缓慢推进，增加电影感。
好：人物把录音笔放上桌时，摄影机小幅推进，使红色录音灯进入可读范围。
```

### 运动路径必须可执行

记录：

- 起始机位。
- 结束机位。
- 路径。
- 速度变化。
- 跟随目标。
- 是否跨轴。
- 遮挡窗口。
- 焦点策略。

“镜头自由穿梭”“镜头丝滑环绕所有人”不是路径。

### 不让运镜替代切镜

如果摄影机运动依次承担建立空间、人物近景、道具特写和门外揭示，它已经包含多个镜头意图。默认拆镜。

## 连续长镜头例外

长镜头是显式高级模式，不是逃避拆镜的容器。

只有满足以下条件时启用：

- 用户明确要求连续长镜头或一镜到底。
- 连续时间和空间具有叙事价值。
- 调度、焦点、灯光、声音和路线已经规划。
- 景别变化有连续摄影路径，不是文字中的隐含剪辑。
- 已提供失败降级方案。

Shot Card 必须增加：

```yaml
long_take_exception: true
long_take_exception_reason:
start_composition:
end_composition:
dominant_shot_size:
shot_size_transitions:
camera_path:
blocking_path:
focus_plan:
axis_crossing_plan:
hidden_stitch_points:
fallback_shot_intent_ids:
```

### 长镜头允许景别变化

可以从全景推进到近景，但必须：

- 写清每个连续阶段。
- 写清人物与摄影机相对运动。
- 不把中间切点隐藏在“然后镜头切到”中。
- 给出无法完成时的原子拆分方案。

### 隐藏拼接

遮挡、甩镜、黑场或快速运动可作为拼接点，但若实际使用拼接，成片数据应记录为多个 Media Asset 或 Timeline Clip。不要在数据层谎称它是单个未经剪辑的 Take。

## 原子化工作流

按以下步骤冻结 Shot：

1. 读取 Beat、Blocking Plan、Coverage Need 和 Shot Intent。
2. 锁定主要叙事目的。
3. 锁定主要注意力中心。
4. 选择一个主导景别。
5. 写出首帧可见状态。
6. 写出一个主要行动单元。
7. 写出尾帧可见状态。
8. 指定一个主要摄影运动或固定机位。
9. 标记切入和切出动机。
10. 检查动作相位和相邻镜头连续性。
11. 检查事件时间轴能否容纳动作、对白、反应和运镜。
12. 检查所选制作方式的风险。
13. 若出现拆镜触发器，建立新的 Shot Intent，不在描述中偷偷塞入第二镜。
14. 若使用长镜头例外，填满例外字段并生成原子降级方案。

## 高频好坏例

### 多景别

```text
坏：
全景显示办公室，她走进来，镜头切近景看她皱眉，再特写桌上的辞职信。

好：
Shot 1 全景：她推门进入办公室，在桌前停下。
Shot 2 近景：她看见桌上的辞职信，眉头收紧。
Shot 3 插入：辞职信上的签名栏已经有人代签。
```

```text
坏：
中景跟拍他走到窗边，随后推进成眼睛特写。

好：
默认拆为中景跟拍和眼部特写。
例外：若明确启用长镜头，写明轨道路径、焦点变化和从中景到特写的连续距离。
```

### 隐含剪辑

```text
坏：
她打开门，镜头来到门外看见警察。

好：
Shot 1 室内中景：她拉开门，视线停在门外。
Shot 2 门外反打：两名警察站在门口。
```

```text
坏：
镜头围绕桌子一圈，再从桌下特写录音笔。

好：
Shot 1 双人中景：两人隔桌对峙。
Shot 2 桌下插入：录音笔红灯闪烁。
```

### 动作过载

```text
坏：
她冲进屋、甩下包、翻抽屉、找到戒指、打电话、哭着跑出去。

好：
Shot 1 她冲进屋，把包甩到沙发上。
Shot 2 她拉开抽屉，翻出戒指盒。
Shot 3 插入：戒指盒是空的。
Shot 4 她拨通电话，把空盒举到镜头前。
```

```text
坏：
他一边打电话一边做饭、看监控、躲开飞来的杯子并关掉燃气。

好：
优先拆出监控信息和飞杯动作；保留一个当前主要危险行动。
```

### 关系镜头

```text
坏规则：
画面有两个人，所以必须拆成两个镜头。

好：
双人中景中，她把钥匙递出，他后退半步不接。两人共同完成“交接失败”这一关系行动。
```

```text
坏：
A 说话，B 反应，门外 C 偷听，桌下手机同时录像。

好：
把主对话、C 偷听和手机录像分别设计为可控 Coverage。
```

### 注意力转移

```text
坏：
近景拍经理说话，随后观众注意到后景秘书偷偷删除文件。

好：
若删除文件是关键事件，建立独立 Shot 或明确焦点转移并评估是否仍属于同一目的。
```

```text
坏：
人物说完秘密，镜头又扫到所有人的反应。

好：
为改变局势的关键反应建立独立反应 Shot；非关键群体反应可保留在同一关系镜头。
```

### 开始和结束状态

```text
坏：
开始：她很紧张。
动作：她看合同。
结束：她决定反击。

好：
开始：合同合上，签字笔在经理手边。
动作：她翻到末页，把签字笔拿到自己面前。
结束：合同停在伪造签名页，签字笔在她右手。
```

```text
坏：
他拿起杯子。

好：
开始：杯子在桌面右侧。
动作：他右手握住杯把，抬到胸口高度。
结束：杯子离开桌面，右手持杯，左手仍按住文件。
```

### 运镜

```text
坏：
镜头推进、环绕、升高，再俯冲到桌上证据。

好：
固定中景拍人物将证据放上桌；另设插入镜头读取证据。
```

```text
坏：
手持晃动制造紧张感。

好：
人物后退撞到墙时，手持摄影机同步向后半步并产生一次可控震动。
```

### 时空

```text
坏：
同一镜头中她离开公司，回到家后打开信。

好：
公司出口 Shot、转场 Shot、家中开信 Shot 分开。
```

```text
坏：
他回忆童年，镜头直接变成童年画面，仍使用同一 Shot ID。

好：
主观回忆建立新 Shot，并标记时间层级和来源。
```

### 长镜头

```text
坏：
一镜到底：先拍街道全景，再切人物近景，然后切手机特写。

好：
连续长镜头：摄影机从街道全景沿人行道跟随人物，人物停下并把手机举到胸前；摄影机继续推进到中近景。手机内容不承担必须读取的信息。
```

```text
坏：
用“无缝转场”掩盖地点跳跃。

好：
若通过遮挡拼接跨地点，记录两个 Shot 或 Timeline Clip，并明确转场设计。
```

## 硬失败

出现以下任一项，退回原子分镜阶段：

- 同一 Shot 写入多个互相独立的景别，且未启用长镜头例外。
- 描述中出现“切到、再切、反打、转为特写”等内部剪辑，却仍使用一个 Shot ID。
- 同一 Shot 无过渡地改变地点、时间或现实/回忆层级。
- 没有明确主要主体、开始状态、行动单元或结束状态。
- 一个镜头承担多个可独立成立的叙事结果。
- 主导景别明显跨级，既不拆镜也不填写长镜头例外。
- 摄影机路径在物理上不连续，或穿墙、瞬移、无因改变方向。
- 动作、对白、反应和运镜无法在 capture/generation duration 内完成。
- Shot 与 Blocking Plan 的 Mark、轴线、视线或道具状态冲突。
- Shot 不对应任何 Beat、Coverage Need 或授权新增内容。
- 通过增加抽象解释掩盖画面不能传递的信息。
- 关系镜头中多个独立主体同时完成无共同因果的关键动作。
- 使用隐藏拼接却把多个素材伪装成一个未经剪辑的 Take。

## 软警告

以下情况先评估风险，不直接失败：

- 同一镜头出现多个主体，但主次和共同因果清楚。
- 情绪词附着在明确可执行动作后。
- 使用同级范围内的小幅推拉或重新构图。
- 对焦从前景转到后景，但叙事目的保持一致。
- 动作链包含准备、执行和即时结果。
- 双人关系镜头同时包含听者即时反应。
- 使用两个摄影运动，但第二个只是保持构图的修正。
- Shot 接近时长或生成复杂度上限。
- 主导景别边界存在争议，但主体比例变化较小。
- 使用长镜头例外，字段完整但制作成本较高。
- 依赖极细小道具或微表情传递非关键细节。
- 为剪辑安全增加了轻微动作重叠。

## Shot Card 检查表

冻结正式 `shot_id` 前逐项确认：

- [ ] 已关联 Scene、Beat、Coverage Need 和 Provisional Shot Intent。
- [ ] 已记录原文来源或授权新增。
